    [언리얼4] 익스포트/임포트 작업노트 / 일상/리뷰

    2015. 12. 20. 23:17

    http://eeeverrrich.blog.me/220574003168

    번역하기 전용뷰어 보기

    * 언리얼4 작업을 기록하기 위한 글입니다. 

    * 추가/수정이 될 수 있습니다.



    * Unreal에서 만든 환경(environments) 전체를 하나의 개체로 Export 하고 싶을 때

    - Editor Preferences > Experimental > Actor Merging(병합) 을 선택하면 에디터가 재실행된다.

    - 레벨에 있는 액터들을 선택하고 오른쪽 마우스 클릭을 하면 Merge 옵션이 보인다.

    - 액터들을 병합시키면, 콘텐트 브라우저에 병합된 개체로된 새로운 에셋이 생성된다.

    - 이것을 FBX로 익스포트하면 된다.


    * 확장자가 TIF인 텍스쳐를 언리얼에 임포트 했을 때, 해당 메테리얼에 적용이 되지 않았음

    - TGA 확장자로 변경한 후, 콘텐트 브라우저로 직접 임포트 하였더니 적용이 되었다.


    * 메테리얼들을 다른 프로젝트 파일로 익스포트하고 싶을 때

    - 방법1) 에셋을 선택하고 오른쪽 마우스 클릭 > Migrate(이동)

    - 방법2) .uasset 파일을 복사해서 옮기고 싶은 프로젝트의 폴더에 붙임, 경로/링크가 깨질 수 있음. 비추.


    * FBX 콘텐츠 파이프라인

    https://docs.unrealengine.com/latest/KOR/Engine/Content/FBX/index.html

     

    * minimum bone size error

    - 모델 크기가 너무 작으면, auto-physics asset generation이 하는 경고.

    - 모델 크기를 키우거나, 에셋을 더 작게 셋팅하여 재계산하도록 한다.


    * FBX 임포트 옵션

    - https://docs.unrealengine.com/latest/KOR/Engine/Content/FBX/ImportOptions/index.html

    - Normal Import Method 노멀 임포트 메소드 : 노멀 처리방식 조절

     

    - 언리얼에서 노멀을 자동 계산하면 메시의 모서리가 엇나감. 싫으면 노멀 임포트 선택.

    글보내기 기능을 제한합니다.

    네이버는 블로그를 통해 저작물이 무단으로 공유되는 것을 막기 위해, 저작권을 침해하는 컨텐츠가 포함되어 있는 게시물의 경우 글보내기 기능을 제한하고 있습니다.
